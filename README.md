# aitoon-page-director

> 스토리와 미드저니 sref 스타일 이미지를 던지면, **연출된 페이지 만화를 GPT(나노바나나)로 뽑는 "복붙용 프롬프트"** 를 만들어 주는 Claude 스킬.

"스토리를 던지면 컷만화 한 장 뚝딱"이 만드는 건 대개 **글+그림 덩어리**입니다. 이 스킬은 그걸 넘어, 컷의 크기·앵글·칸 모양 같은 **연출**과 캐릭터·무기·방향의 **일관성**, 그리고 미드저니 sref **화풍**을 1차 프롬프트에 전부 박아서 — 운 좋은 한 장이 아니라 **"이렇게 하면 매번 되는"** 페이지 만화를 만듭니다.

---

## 핵심 아이디어

좋은 페이지 만화는 운이 아니라 **재현 방법**입니다. 그래서 이 스킬은 연출 명세의 모든 통제를 유저가 보지 않는 1차 프롬프트 안에 녹여 한 방 성공률을 끌어올립니다.

이 스킬을 쓰는 사람은 **만화는 만들고 싶지만 과정은 귀찮은 사람**입니다. 그래서 씬 분할·컷 설계·페이지 배분·스타일 토큰 추출 같은 판단은 전부 내부에서 처리하고, 유저 눈에는 **복붙할 프롬프트만** 떨어집니다. (레이아웃 시안 같은 중간 산출물은 보여주지 않습니다.)

---

## 어떻게 동작하나

```
스토리 던지기  →  스타일 이미지(sref) 첨부  →  프롬프트 복붙해서 생성  →  (선택) 결과 올려서 수정
```

유저가 손대는 접점은 이 4개뿐입니다. 나머지는 스킬이 알아서 합니다.

| 단계 | 하는 일 |
|------|---------|
| 1. 스토리 | 받아서 (내부) 씬·컷 구성 + 4~6페이지 배분 + 페이지별 감정 비트 |
| 2. 스타일 | 미드저니 sref 그리드/코드 → 스타일 토큰 7분해 + 캐릭터 경로 판정 |
| 3. 캐릭터 ref | **sref 코드 있으면 미드저니로(정석) / 없으면 GPT로(폴백)** 캐릭터 생성 프롬프트 |
| 4. 페이지 프롬프트 | 연출 3대 레버 + 일관성 + 스타일 앵커 + 클린 원고를 박은 페이지별 복붙 프롬프트 |
| 5. 사용 안내 | 첨부 순서·비율·GPT 사용법 — **여기까지가 공식 끝** |
| 6. (선택) 수정 | 결과 이미지를 올리면 그 페이지 프롬프트만 진단·수정 (자동 반복 루프는 안 돔) |

---

## 캐릭터는 미드저니, 페이지는 GPT (하이브리드)

```
sref 코드 보유  → 캐릭터 = 미드저니(--sref 코드, 정석)  ┐
                                                       ├→ 페이지 = GPT(캐릭터 이미지 + 그리드 첨부)
그리드 이미지만 → 캐릭터 = GPT(그리드 첨부, 폴백)        ┘
```

캐릭터는 **미드저니로 원본을 잡아** 그걸 GPT 페이지에 ref로 넣는 게 정석입니다. 그래야 GPT 특유의 평평한 화풍에서 벗어나 sref 원본 화풍이 또렷이 묻습니다. 페이지(여러 컷 연출)는 미드저니가 약하므로 **GPT로 생성**합니다 — 캐릭터만 분기합니다.

## 설치

1. `aitoon-page-director.skill` 파일을 받습니다. (또는 이 폴더를 그대로 스킬로 설치)
2. Claude 데스크톱 **설정 > Capabilities** 에서 스킬을 설치합니다.
3. "페이지 만화 만들어줘" / "이 스토리로 컷만화 프롬프트 뽑아줘" 같은 말로 실행됩니다.

> 페이지형 컷만화(트랙 A) 전용입니다. 인스타툰 4컷·세로 스크롤 웹툰은 다른 스킬을 쓰세요.

## 하는 것 / 안 하는 것

**한다** — 씬·컷 내부 구성 · 스타일 토큰 추출 · 캐릭터 ref 프롬프트 · 페이지별 복붙 프롬프트 · 사용 안내 · (선택) 이미지 보고 프롬프트 수정.

**안 한다** — 이미지 직접 생성(GPT·미드저니 생성은 유저 몫) · 자동 v1→v2→v3 반복 루프 · 말풍선 합성·편집(클린 원고만) · 트랙 B 세로 웹툰 · 레이아웃 시안 노출.
